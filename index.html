Kefir.js

Kefir — is an FRP (functional reactive programming) library for JavaScript inspired by Bacon.js and RxJS with focus on high perfomance and low memory usage.
It has nice Bacon.js like API and RxJS's performance / memory usage characteristics.

Kefir has GitHub repository, where you can send pull requests, report bugs, and have fun reading source code.

Download (0.1.10)

kefir.js ~ 30 kb
kefir.min.js ~ 5 kb (when gzipped)

You can also get edge version from GitHub, but use it at your own risk

NPM

npm install kefir

Bower

bower install kefir

As you can see docs aren't finished yet, but you can already get some info here. If you have questions feel free to talk to me on twitter, github issues or via email.

Static methods (Kefir.—)

fromPollstream = Kefir.fromPoll(interval, fn, [context], *arg)
Returns a Stream. When stream has subscribers, fn being polled with given interval (in milliseconds). Values returned by fn will be delivered to subscribers. fn can return special values. When fn returns Kefir.END stream ends as well as polling of fn. 

var values = [1, 2, 3, Kefir.END];
var stream = Kefir.fromPoll(100, function(){
  return values.shift();
});
stream.log();
=> [FromPollStream] <value> 1
=> [FromPollStream] <value> 2
=> [FromPollStream] <value> 3
=> [FromPollStream] <end>

fromBinderstream = Kefir.fromBinder(subscribe, [context], *arg)
Returns a Stream. The parameter subscribe is a function that accepts a sink which is a function that your subscribe funtion can "push" values to. 

var stream = Kefir.fromBinder(function(sink){
  setTimeout(function(){
    sink(1);
    sink(2);
    sink(Kefir.error('oops'));
    sink(3);
    sink(Kefir.END);
  }, 0);
});
stream.log();
=> [FromBinderStream] <value> 1
=> [FromBinderStream] <value> 2
=> [FromBinderStream] <error> oops
=> [FromBinderStream] <value> 3
=> [FromBinderStream] <end>

You can pass special values to sink.

The subscribe function may return a function. Let's call that function unsubscribe. This function is called when the last listener removes from stream. You can use unsubscribe function to unsubscribe from your original source of values, for example jQuery events listener. 

var stream = Kefir.fromBinder(function(sink){
  $('#button').on('click', sink);
  return function(){
    $('#button').off('click', sink);
  }
});
stream.log();
=> [FromBinderStream] jQuery.Event {type: "click"...}
=> [FromBinderStream] jQuery.Event {type: "click"...}

The subscribe function is called when the first stream listener is added, and the unsubscribe function is called only after the last listener has been removed. The subscribe-unsubscribe cycle may of course be repeated indefinitely, so prepare for multiple calls to the subscribe function.

Kefir.fromBinder() is a base building block upon which you can build your own custom Stream with any original source of values. Also consider using bus if you want to build custom stream.
